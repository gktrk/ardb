ARDB Manual


Card Browser window (main window)


Each tab in the window is a different browser.

You start with two of them (one for the crypt, one for the library) and ALL cards in the browsers.

The "Select", "Add" and "Remove" buttons are filters. First, use "Select" to focus your search. Then you can use "Add" and "Remove", which should be self-explanatory.

There's only one search per tab. To have multiple simultaneous searches, click the various "New [whatever] browser" in the "Browser" menu.

The green boxes that appear along the top border are reminders of the filters you've applied. They're like the stack of filters in Photoshop or the GIMP. The order of these filters is important : they're applied from left to right. You can manipulate them with a right-click which brings a popup menu.

Left clicking the colum names of the card list allows you to sort the results.

Typing a card's name in the "Find" textfield will highlight this card in the list. This "Find" function has spellchecking which allows for one typo.

Right clicking a card in the list pops up a menu. Double clicking it adds this card to the current deck.

Multiple selection of cards or filters is allowed.

Deck Builder window

There are 3 tabs : deck information, crypt and library.

Deck information is simple, just fill the form.

The crypt tab allows you to pick vampires to add in your deck. They'll appear in the bottom list. Before adding each vampire, you may also choose the vampire's edition and "Advanced" status.

As a visual clue, some vampires in your crypt list may show up in red. This is because they violate the crypt grouping rule. Right clicking vampires in this list brings up a popup menu.

The Happy Families calculator is located in the top right corner, and may serve as a deckbuilding help to good cardflow.

Instead of the Happy Families calculator, you may opt to display the vampire's cardtext, useful if you're not familiar with the cards.

The library tab allows you to add library cards for your deck. Your deck is displayed as a tree on the right side. Right clicking cards in this tree brings up a popup menu.

As a deckbuilding help, the bottom left area may display a library overview of how many cards of each kind are being used, or you may choose to use this space as a card text reminder.

The File menu of the Deck Builder window has all the classical file operations (open, save, export).
The Tools menu includes Merge Deck (use this for including pre built modules in your current deck), and Draw Simulator (shuffles your deck and simulates your first hand and uncontrolled region).

FAQ, FGA

Q: My deck morphed! Cards are completely random!
A: In previous releases of ARDB, a failed database update (the Anthelios bug for example) would lead to a messed up but functional database. All decks built with such a messed up database are messed up decks. I'm sorry about that. Now ARDB will cancel the changes and revert to a sane database if anything goes wrong during the update.

Q: I want Master cards to be listed before everything else in my deck lists.
A: ARDB now comes with "Masters First" templates (*mf.xsl). Please tweak your ARDB.INI configuration file so that ARDB uses these new templates. It should be very straightforward. A graphical means of doing this might appear in the future, but it's not a priority right now.

Q: I still don't like the way ARDB exports decks.
A: Write your own template then, or tweak the existing ones. For example, changing colors in the HTML export template should be pretty easy if you know a little about HTML and CSS.

Q: Your deckbuilder doesn't work on Windows 95/98.
A: True. ARDB is based on fairly recent technologies, like Unicode. Windows 95/98 don't have proper Unicode support. They're too old. Or they're not good enough, I don't know. Either way, they're no longer supported by Microsoft anyway, so I fail to see why I would support them. You should probably upgrade to a better system... May I suggest MacOS or Linux? :)

Q: There's a bug when...
A: Please submit a proper bug report at https://savannah.nongnu.org/bugs/?func=additem&group=anarchdb
